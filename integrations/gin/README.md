# BarefootJS + Gin Example

This example runs the shared BarefootJS components on Go's [Gin](https://gin-gonic.com/)
(github.com/gin-gonic/gin), demonstrating server-side rendering + client-side hydration with the
`@barefootjs/go-template` adapter.

It is a peer of the `integrations/echo` example — same JSX components
(`integrations/shared/components`), same shared styles, same routes — wired
to a Gin server instead of Echo.

## Quick start

```bash
bun install            # from the repo root (workspace deps)
cd integrations/gin

bun run build          # JSX → dist/templates + dist/client, copy shared styles
bun run dev            # APP_ENV=development BASE_PATH=/integrations/gin go run .
# → http://localhost:8081/integrations/gin
```

In development (`APP_ENV=development`) the server re-parses templates on every
request and serves an SSE endpoint at `/integrations/gin/_bf/reload`, so edits picked up by
`bun run build:watch` reload the browser automatically. In production
(`APP_ENV=production`) templates are parsed once at startup and no dev snippet
is emitted.

## How it works

| Concern | Where |
|---|---|
| Routes + server | `main.go` |
| Page layout (HTML shell) | `defaultLayout` in `main.go` |
| Generated Go struct types (`New*Props`, `*Input`) | `components.go` (regenerated by `bun run build`) |
| Compiled templates / client JS | `dist/` (git-ignored build output) |

Handlers build props with the generated `New<Component>Props(<Component>Input{...})`
constructors and hand them to `bf.Renderer.Render`, which executes the
component's Go template and wraps it in `defaultLayout`.

## URL layout

- `/integrations/gin/` — index with links to every demo
- `/integrations/gin/counter`, `/integrations/gin/toggle`, `/integrations/gin/form`, `/integrations/gin/portal`,
  `/integrations/gin/reactive-props`, `/integrations/gin/conditional-return` — component demos
- `/integrations/gin/todos` / `/integrations/gin/todos-ssr` — interactive todo list backed by an
  in-memory, per-session (`bf_session` cookie) store
- `/integrations/gin/ai-chat` + `/integrations/gin/api/ai-chat` — token-by-token SSE streaming demo
- `/integrations/gin/static/client/*` — compiled client JS
- `/integrations/gin/shared/styles/*` — shared CSS (copied into `dist/shared` by
  `scripts/copy-shared.ts`)

## Tests

```bash
bun run test:e2e       # Playwright; boots the Go server automatically
```

The specs in `e2e/` re-use the shared suites in `integrations/shared/e2e`, so
the same user-facing behaviour is verified across every adapter.
