# BarefootJS + Chi Example

This example runs the shared BarefootJS components on Go's [Chi](https://go-chi.io/)
(github.com/go-chi/chi/v5), demonstrating server-side rendering + client-side hydration with the
`@barefootjs/go-template` adapter.

It is a peer of the `integrations/echo` example ‚Äî same JSX components
(`integrations/shared/components`), same shared styles, same routes ‚Äî wired
to a Chi server instead of Echo.

## Quick start

```bash
bun install            # from the repo root (workspace deps)
cd integrations/chi

bun run build          # JSX ‚Üí dist/templates + dist/client, copy shared styles
bun run dev            # APP_ENV=development BASE_PATH=/integrations/chi go run .
# ‚Üí http://localhost:8082/integrations/chi
```

In development (`APP_ENV=development`) the server re-parses templates on every
request and serves an SSE endpoint at `/integrations/chi/_bf/reload`, so edits picked up by
`bun run build:watch` reload the browser automatically. In production
(`APP_ENV=production`) templates are parsed once at startup and no dev snippet
is emitted.

## How it works

| Concern | Where |
|---|---|
| Routes + server | `main.go` |
| Page layout (HTML shell) | `defaultLayout` in `main.go` |
| Generated Go struct types (`New*Props`, `*Input`) | `components.go` (regenerated by `bun run build`) |
| Compiled templates / client JS | `dist/` (git-ignored build output) |

Handlers build props with the generated `New<Component>Props(<Component>Input{...})`
constructors and hand them to `bf.Renderer.Render`, which executes the
component's Go template and wraps it in `defaultLayout`.

## URL layout

- `/integrations/chi/` ‚Äî index with links to every demo
- `/integrations/chi/counter`, `/integrations/chi/toggle`, `/integrations/chi/form`, `/integrations/chi/portal`,
  `/integrations/chi/reactive-props`, `/integrations/chi/conditional-return` ‚Äî component demos
- `/integrations/chi/todos` / `/integrations/chi/todos-ssr` ‚Äî interactive todo list backed by an
  in-memory, per-session (`bf_session` cookie) store
- `/integrations/chi/ai-chat` + `/integrations/chi/api/ai-chat` ‚Äî token-by-token SSE streaming demo
- `/integrations/chi/static/client/*` ‚Äî compiled client JS
- `/integrations/chi/shared/styles/*` ‚Äî shared CSS (copied into `dist/shared` by
  `scripts/copy-shared.ts`)

## Tests

```bash
bun run test:e2e       # Playwright; boots the Go server automatically
```

The specs in `e2e/` re-use the shared suites in `integrations/shared/e2e`, so
the same user-facing behaviour is verified across every adapter.
